# Case Study 1

## Flyrank AI Internship

### The Problem

I wanted to become more productive with AI than simply asking ChatGPT random questions. My goal was to build repeatable workflows that improve how I research, write prompts, debug applications, and document projects.

### What I Did

Throughout the internship I completed weekly assignments focused on prompt engineering, AI workflows, documentation, and structured problem solving.

I also applied these techniques to my own software projects and backend learning.

### Outcome

I now have reusable AI workflows that reduce repetitive work while keeping me in control of technical decisions.

The internship also improved how I communicate technical work and document engineering decisions instead of only writing code.

# Case Study 2

## Salamat

### The Problem

Managing appointments between patients, doctors, and administrators involves many small workflows that easily become inconsistent.

The challenge wasn't just displaying data—it was designing a system that prevents mistakes while remaining easy to use.

### What I Did

I designed and implemented features across multiple user roles including patients, doctors, and administrators.

Some examples include:

- appointment scheduling
- archived consultations
- authentication improvements
- validation rules
- clinic management
- doctor consultation hours
- UI improvements
- bug fixing
- feature planning

Before implementing features, I broke problems into user flows and edge cases to reduce future maintenance.

### Outcome

The project evolved into a more reliable management system with cleaner workflows and a better user experience.

It also strengthened my ability to think beyond individual pages and consider the entire product.

# Case Study 4

## Hospital Management REST API

### The Problem

Healthcare systems require secure access control because different users need different permissions.

The challenge was designing an API that remains organized while enforcing security.

### What I Did

I built a REST API featuring:

- JWT authentication
- role-based authorization
- CRUD operations
- validation
- protected routes
- modular backend architecture

Instead of writing endpoints one by one, I organized the project around maintainability and separation of concerns.

### Outcome

The project strengthened my backend engineering skills and gave me experience implementing authentication and authorization patterns used in real applications.
